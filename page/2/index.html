How to install Refinery CMS to Heroku

Install

gem install refinerycms
refinerycms myapp --heroku

heroku create myrefinerycms

Create Heroku Postgres DB

git push heroku master

Edit config/application.rb and add:

config.assets.initialize_on_precompile = true

Run:

heroku run rake db:migrate
heroku run rake db:seed

Visit https://myrefinerycms.herokuapp.com for your website.

To configure your website, access: https://myrefinerycms.herokuapp.com/refinery

To configure company’s name, edit config/initializers/refinery/core.rb, add:

config.site_name = "NCELP"

To override your first view:

rake refinery:override view=refinery/pages/show

Now edit app/views/refinery/pages/show.html.erb, put the content as:

<section id="body">
  <%= raw @page.content_for(:body) %>

Ubuntu

Install and configure zsh to be git friendly

Install zsh

Install zsh:

sudo apt-get install zsh
sudo usermod -s /bin/zsh $(whoami)

Reboot

Install zsh plugins

Install Powerlevel9k theme:

sudo apt-get install zsh-theme-powerlevel9k
echo "source /usr/share/powerlevel9k/powerlevel9k.zsh-theme" >> ~/.zshrc

Install zsh-syntax-highlighting:

sudo apt-get install zsh-syntax-highlighting
echo "source /usr/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh" >> ~/.zshrc

Install Oh-My-ZSH:

sh -c "$(wget https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh -O -)"

Choose a theme and change .zshrc, the themes and their look can be found here:

https://github.com/robbyrussell/oh-my-zsh/wiki/themes

make sure copy all environment variables from .bashrc/.bash_profile to ~/.zshrc

Fix zsh cursor position problem:

In some version of Ubuntu, the cursor is a character behind, to fix this problem:

Create a sh script and run it:

#!/bin/sh
DIR="$HOME/.oh-my-zsh/custom/themes"
mkdir -p $DIR
cd $DIR
sed "s/➜/→/g;s/✗/×/g" $HOME/.oh-my-zsh/themes/robbyrussell.zsh-theme > robbyrussell.zsh-theme

git clone git@github.com:yf508/yorkshire-dictionary.git

cd yorkshire-dictionary

heroku create --region eu

In Heroku dashboard, under the application you just created, add a Heroku Postgres Add-on

heroku config:set RACK_ENV=heroku RAILS_ENV=heroku --remote heroku

heroku config:set RAILS_MASTER_KEY=YOUR_KEY

heroku run rails db:schema:load

git remote add origin git@github.com:digital-york/yorkshire-dictionary.git

Create docker-compose.yml

version: '3'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - .:/usr/src/railsdemo
    command: bash -c "rm -f tmp/pids/server.pid && bundle exec rails s -p 3000 -b '0.0.0.0'"

Then run:

docker-compose up --build

Useful commands

Image

List of images

docker images

Remove an image

docker rmi xxxx

Delete all images and their dependent images

docker rmi -f $(docker images -q)

Container

Start a container

docker-compose up

List of containers

docker ps (ONLY running containers)
docker ps -a (all containers)
docker-compose ps

Stop all running containers

docker stop $(docker ps -aq)

Remove all containers

docker rm $(docker ps -aq)

A script to remove all containers and images:

#!/bin/bash
# Stop all running containers
docker stop $(docker ps -aq)

# Delete all containers

# Delete all images and their dependent images
docker rmi -f $(docker images -q)

Service

Show logs of a service

docker-compose logs web

Manage a service

docker-compose [start|stop|kill|restart|pause|unpause|rm] SERVICE

SSH to a container

docker-compose run web bash

How to add a survey window before download file

First, generate a new model

Run this command to generate a new Downloader:

rails g hyrax:work Downloader

Add additional downloader metadata

Add app/models/concerns/oasis/downloader_metadata.rb:

module Oasis
  module DownloaderMetadata extend ActiveSupport::Concern
    included do
      property :time, predicate: ::RDF::URI.new('https://oasis-database.org/ontologies/oasis#downloader_time'), multiple: false do |index|
        index.as :stored_searchable
      end

      # ......
  end
end

Also, edit app/models/downloader.rb to add:

include ::Oasis::DownloaderMetadata

Setup relationships

Define the relationship between Summary and Downloader:

In Summary,

class Summary < ActiveFedora::Base
  # ......
  has_many :downloaders, dependent: :destroy, class_name: 'Downloader', :autosave => true

  # ......
end

In Downloader,

class Downloader < ActiveFedora::Base
  belongs_to :summary, :autosave => true, predicate: ActiveFedora::RDF::Fcrepo::RelsExt.isPartOf

  # ......
end

The standard RoR form (not Hyrax form)

Since the downloader survey form does not require authentication, the Hyrax work based form (e.g. the default downloader form is not appropirate).

In here, we use a standard RoR form to get users’ input and create Downloader object.

generate a standard RoR form/controller

rails g controller Survey new

In Survey Controller, can do sth. like:

def submit
  begin
    summary_id         = params["summary_id"]
    summary_fileset_id = params["summary_fileset_id"]

    d = Summary.find(summary_id).downloaders.new
    d.downloader_status      = params["survey"]["status"] unless params["survey"]["status"].nil?
    d.downloader_email       = params["email"]            unless params["email"].nil?
    d.downloader_affiliation = params["institution"]      unless params["institution"].nil?
    d.downloader_contable    = params["contable"]         unless params["contable"].nil?
    d.downloader_cty         = params["country"]          unless params["country"].nil?
    d.save!

    redirect_to '/survey/saved?survey_done=yes&summary_id=' + summary_id.to_s + '&downloader_id=' + d.id.to_s + '&fileset_id=' + summary_fileset_id.to_s
    rescue
    redirect_to '/survey/error'
  end
end

Adding a new static page in Hyrax

Hyrax provides a nice feature to let users define the content of static pages as content blocks. But sometimes, users may want to add a new page and keep updating the content of that page by non-technical people. Here is how:


Image source: pixabay.com

Register the new page in ContentBlock

Edit file: app/models/content_block.rb

class ContentBlock < ActiveRecord::Base
  NAME_REGISTRY = {
    #......
    activities: :activities_page
  }.freeze

  class << self
    #......
    def activities_page
       find_by(name: 'activities_page') ||
          create(name: 'activities_page', value: default_activities_text)
    end

    def activities_page=(value)
       activities_page.update(value: value)
    end

    #......
    def default_activities_text
      ERB.new(
          IO.read(
              Rails.root.join('app', 'views', 'hyrax', 'content_blocks', 'templates', 'activities.html.erb')
          )
      ).result
    end
  end  
end

Register the new page in page controller

Copy app/controllers/hyrax/pages_controller.rb from Hyrax to your APP, and add the following information:

module Hyrax
  class PagesController < ApplicationController
    # ......

    private
      def permitted_params
          params.require(:content_block).permit(:about,
                                                :agreement,
                                                :help,
                                                :terms,
                                                :activities)
      end
  end
end

Add a page for default content

Create a new file and put any content you like: app/views/hyrax/content_blocks/templates/activities.html.erb

Make changes to UI

Copy file app/views/hyrax/pages/_form.html.erb from Hyrax to your APP and edit:

<div class="panel panel-default tabs">

    <!-- ...... -->

    </li>
  </ul>

  <div class="tab-content">
      <!--......-->
      <div id="activities" class="tab-pane">
        <div class="panel panel-default labels">
          <%= simple_form_for ContentBlock.for(:activities), url: hyrax.page_path(ContentBlock.for(:activities)) do |f| %>
              <div class="panel-body">
                <div class="field form-group">
                  <%= f.label :activities %><br>
                  <%= f.text_area :activities, value: f.object.value, class: 'form-control tinymce', rows: 20, cols: 120 %>
                </div>
              </div>
              <div class="panel-footer">
                <%= link_to t(:'hyrax.pages.cancel'), hyrax.admin_admin_sets_path, class: 'btn btn-default pull-right'%>
                <%= f.button :submit, class: 'btn btn-primary pull-right'%>
              </div>

        </div>
      </div>

Define SolrDocument

The attributes should be exposed in SolrDocument. By this way, these attributes are accessable directly from SolrDocument in the view later.

module Oasis
  module SolrDocument extend ActiveSupport::Concern
  module Solr
    class Array
      # @return [Array]
      def self.coerce(input)
        ::Array.wrap(input)
      end
    end

    class String
      # @return [String]
      def self.coerce(input)
        ::Array.wrap(input).first
      end
    end

    class Date
      # @return [Date]
      def self.coerce(input)
        field = String.coerce(input)
        return if field.blank?
        begin
          ::Date.parse(field)
        rescue ArgumentError
          Rails.logger.info "Unable to parse date: #{field.first.inspect}"
        end
      end
    end
  end

  included do
    attribute :title_of_summary, Solr::String, solr_name('title_of_summary')
    attribute :summary_writer, Solr::Array, solr_name('summary_writer')
    attribute :summary_writer_all, Solr::String, solr_name('summary_writer_all')
    # ......
  end  
  end
end
