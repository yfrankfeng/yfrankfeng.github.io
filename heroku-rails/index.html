Heroku

Install and configure Rails application to Heroku

Install heroku cli

sudo snap install --classic heroku

Make sure add ‘/snap/bin’ into PATH and re-login again. Type command:

heroku --version

Setup Rails application in Heroku

In the following, I will use A forked version of Yorkshire Historical Dictionary, the notes below are mainly taken from the README from the original project. 

git clone git@github.com:yf508/yorkshire-dictionary.git

cd yorkshire-dictionary

heroku create --region eu

In Heroku dashboard, under the application you just created, add a Heroku Postgres Add-on

Then go back to Ubuntu terminal, run:

heroku config:set RACK_ENV=heroku RAILS_ENV=heroku --remote heroku

heroku config:set RAILS_MASTER_KEY=YOUR_KEY

git push heroku

Load data:

heroku run rails db:schema:load

heroku run rails yhd:import

Work as a collaborator

Once received invitation as a collaborator, run:

heroku git:clone -a yhd

then link to github repository:

git remote add origin git@github.com:digital-york/yorkshire-dictionary.git

tips

SSH to heroku dyno:

heroku run bash

Problems & solutions

Cannot run heroku container:login while deploying a dockerized app

The error message is:

Error saving credentials: error storing credentials - err: exit status 1, out: Cannot autolaunch D-Bus without X11 $DISPLAY
▸ Login failed with: 1

Solution

sudo apt install gnupg2 pass
gpg2 --full-generate-key
gpg2 -kgpg2 -k
pass init " the id from above command"

Then run:

heroku container:login
