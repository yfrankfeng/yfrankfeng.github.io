OAuth login upgrade while Google+ shutdown

To test Google OAuth 2, try OAuth 2.0 Playground: https://developers.google.com/oauthplayground/

The following URLs seem working:

https://www.googleapis.com/oauth2/v2/userinfo
https://openidconnect.googleapis.com/v1/userinfo

OLD Plus+ API returns:

https://www.googleapis.com/plus/v1/people/me

{
“kind”: “plus#person”,
“displayName”: “XXXXXX”,
“name”: {
“givenName”: “XXXXXX”,
“familyName”: “XXXXXX”
},
“language”: “en_GB”,
“isPlusUser”: true,
“url”: “https://plus.google.com/+XXXXXX",
“gender”: “male”,
“image”: {
“url”: “https://lh6.googleusercontent.com/XXXXXX.jpg",
“isDefault”: false
},
“etag”: “\”XXXXXX\””,
“verified”: false,
“circledByCount”: 0,
“id”: “XXXXXX”,
“objectType”: “person”
}

In OAuth 2.0 Playground, the following call returns:

https://www.googleapis.com/oauth2/v2/userinfo

{
“picture”: “https://lh6.googleusercontent.com/XXXXXX.jpg",
“name”: “XXXXXX”,
“family_name”: “XXXXXX”,
“locale”: “XXXXXX”,
“gender”: “male”,
“link”: “https://plus.google.com/+XXXXXX",
“given_name”: “XXXXXX”,
“id”: “XXXXXX”
}

But in the Java code, it returns:
{
“id”: “XXXXXX”,
“email”: “XXXXXX”,
“verified_email”: true,
“name”: “XXXXXX”,
“given_name”: “XXXXXX”,
“family_name”: “XXXXXX”,
“picture”: “https://lh6.googleusercontent.com/XXXXXX"
}

Since the returned JSON format has been changed, so the processing code is slightly different, e.g.:

// The OLD code working with Google+ has been commented out

if (obj.getString(“email”)!=null)
oauthUser.setEmail(obj.getString(“email”));
oauthUser.setId(OAuth2Manager.getOAuth2Url(OAuth2Manager.OAUTH2_PROVIDER_GOOGLE, obj.getString(“id”))); // oauth2_google_GOOGLE_USER_ID
// oauthUser.setFirstName(obj.getJSONObject(“name”).getString(“givenName”));
oauthUser.setFirstName(obj.getString(“given_name”));
//oauthUser.setSurName(obj.getJSONObject(“name”).getString(“familyName”));
oauthUser.setSurName(obj.getString(“family_name”));
